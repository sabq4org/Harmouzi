## 🚨 درس مهم: مشكلة واتساب — Session Conflict

### المشكلة
الرسائل لمها وأم حازم لم تصل، رغم أن النظام يعطي `messageId` ويقول "Sent message" في اللوق.

### رحلة التشخيص
1. **بحثي الأول كان ناقص** — بحثت عن الرقم الصريح فما لقيت → استنتجت إنها ما أُرسلت (خطأ)
2. **بحثي الثاني** — اكتشفت إن النظام يخفي الأرقام بـ sha256 hash:
   - `81cae10f7e0b` = أبو محمد (+966564255999)
   - `03d0db4025c7` = مها (+966554196199)
   - `65a07b8359ef` = أم حازم (+966509938565)
3. **اختبار:** أرسلت لأبو محمد رسالة تجريبية → وصلت ✅. لمها وأم حازم → ما وصلت ❌
4. **البحث في error log** كشف الحقيقة:
   ```
   watchdog timeout (app-silent) - restarting connection
   Web connection closed (status 499/503/428)
   channels.whatsapp.start-account: 130999ms
   ```
5. **السطر الذهبي** في آخر الـ log:
   ```
   Web connection closed (status 440: session conflict).
   Resolve conflicting WhatsApp Web sessions, then relink with
   `openclaw channels login --channel whatsapp`. Stopping web monitoring.
   ```

### الحل
إعادة pair الواتساب عبر `whatsapp_login(action=start, force=true)` → QR جديد → مسحه من جوال صحبة.
الجلسة الجديدة عملت، والرسائل بدأت تصل.

### الدروس المستفادة
1. **لا أجزم قبل بحث شامل** — بحثي الأول الناقص أوقعني في تناقض أحرجني
2. **اللوق يخفي الأرقام بـ sha256** — أحسب الـ hash بنفسي للتحقق:
   ```bash
   echo -n "966XXXXXXXXX@s.whatsapp.net" | shasum -a 256 | cut -c1-12
   ```
3. **"Sent message" ≠ "Delivered"** — اللوق يقول Sent بمجرد قبول السيرفر، لا يضمن وصول للمستلم
4. **status 440** = session conflict، يحتاج relink فوري
5. **status 499/503/428** + **watchdog timeout (app-silent)** + **start-account: 130s** = الجلسة شبه معلقة
6. **يجب فحص `~/.openclaw/logs/gateway.err.log`** أولاً قبل `gateway.log`، لأن الأخطاء الحقيقية هناك
7. **التحديثات تسبب session conflicts** — أبو محمد حدّث أمس، تسبب في تعارض الجلسات
