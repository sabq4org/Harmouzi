# 2026-05-30 — السبت

## 🐛 إصلاح: تطبيق سبق iOS لا يحمّل المكونات

### الأعراض
- التطبيق يفتح على iPhone بنجاح
- لكن المكونات (الأخبار، الواجهة) لا تتحمّل
- ظهرت بعد ترحيل الفرونت لـ Cloudflare Pages

### السبب الجذري
**CORS**: backend في Railway كان يرفض طلبات من `capacitor://localhost` (origin الـ WebView في iOS).
- المتصفح يطلب من `https://sabq.org` → CORS مسموح ✅
- التطبيق يطلب من `capacitor://localhost` → CORS مرفوض ❌
- النتيجة: API calls تفشل صامتاً، شاشة بدون محتوى

### التشخيص
الفحص الذي كشف المشكلة:
```bash
# يرجع 200 لكن بدون access-control-allow-origin
curl -sI -H "Origin: capacitor://localhost" "https://api.sabq.org/api/articles"
```

### الحل
أضفت في `server/index.ts` (بعد parse-and-protocol gate) استثناء يسمح بـ:
- `capacitor://localhost` (iOS Capacitor)
- `https://localhost` (Android Capacitor 5+)
- `http://localhost` (Android قديم)
- `ionic://localhost` (Ionic legacy)

```typescript
const isCapacitorWebView =
  (parsed.protocol === 'capacitor:' && parsed.hostname === 'localhost') ||
  (parsed.protocol === 'ionic:' && parsed.hostname === 'localhost') ||
  ((parsed.protocol === 'https:' || parsed.protocol === 'http:') && parsed.hostname === 'localhost');
if (isCapacitorWebView) {
  return callback(null, true);
}
```

### Commit
`70a1a41` — fix(cors): allow Capacitor WebView origins for iOS/Android app

### الدرس للمستقبل
- لما يكون عند سبق تطبيق native يستخدم WebView (Capacitor)، دائماً أختبر CORS من origin الـ WebView:
  - iOS: `capacitor://localhost`
  - Android: `https://localhost`
- الأمان مضمون لأن هذي origins محصورة في bundle موقّع، وأي auth tokens لها على API.
- المتصفحات الـ DevTools ما تظهر هذي المشكلة لأنها بـ origin طبيعي.
